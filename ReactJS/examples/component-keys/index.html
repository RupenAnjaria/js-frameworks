Component keys and node reconciliation in lists
This page visualizes how React reconciles the dom nodes when the underlying list data source is changed. According to the ReactJs documentation, here is how it does the reconciliation:
When React reconciles the keyed children, it will ensure that any child with key will be reordered (instead of clobbered) or destroyed (instead of reused).

Visual Reference

1 -> 1 This animation means React created a new dom node. See how all nodes are recreated when random keys are used (don't use this)
1 -> 5 Color code mismatch. React reused an existing node for the new data item. This behavior will be seen when default keys are used.
